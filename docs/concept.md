# 企画・構想シート（study-pilot） v0.1

## ターゲット（ペルソナ）
- 個人で学習を進める人（特に資格試験や学力試験などの受験者）
- 専門家（講師・家庭教師など）の指導を受けず、自主的に学習を進めている

## 課題（Problem）
- 学習目標を達成するまでの道筋（計画）を立てるのが難しい
- 学習対象をタスクに分解するのが難しく、何から手をつけるべきか分からない
- 学習した内容や進捗状況を自力で管理するのが難しい
- 学習中に詰まったときに質問・相談できる相手がいない
- 単に答えを示すだけでなく、なぜそうなるかを理解したい


## 提供価値（Value Proposition）
- ユーザーの学習目標から逆算して、学習対象をタスクに分解し、完了条件つきで提示する
- 学習タスクの進捗を管理し、達成度を可視化する
- 詰まり箇所に対して、理由・根拠付きでサポートを行う
- 計画・実行・復習・理由解説を一体化させ、「学習の道筋を明確にする」という価値を提供する

## 最終ゴール（Outcome）
- ユーザーが目標としている試験や資格に合格できる
- 計画的かつ効率的に学習を進め、学習内容を定着させる力を身につける
- 学習対象を自律的に分解・管理し、自分だけでも学習を完遂できる状態になる

## 成功指標（KPI）※要検討
- 学習対象をタスクとして分解し、完了条件を設定する
- 完了タスク率（例：全300タスク中250完了）
- 週あたり完了タスク数（進捗速度）
- タスク完了時の正答率や理解度スコア（学習の質）

## MVPスコープ（Must）※未確定
- 学習タスク管理（計画と進捗の登録）
- 詰まり箇所への質問・理由付き解説機能
- 進捗ダッシュボード（完了率・学習日数）

## リスキー仮説と検証方法（RAT）※未確定
- 価値仮説：サポートAIを導入することで学習継続率が向上する
　→ 5人×2週間でD14≥35%なら続行
- 技術仮説：自動出題・理由付き解説の品質が十分である
　→ 人手評価平均4/5以上、20問サンプルで検証
